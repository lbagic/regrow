# Changelog

## Unreleased

- Architecture deepening (pre-Prompt F): trash module now owns the trash mechanism — the Finder-move preview command moved from the planner into `internal/trash` (`PreviewCommand`), so the Phase 2 executor lands behind an existing seam; behavior and plan output unchanged. Golden harness rebuilt: every rule carries a `fixture:` block in its own YAML (files + fake tool-query items + optional host override), each rule scans an isolated fixture home and snapshots to `testdata/golden/<rule-id>.golden`, and coverage is enforced — a rule whose fixture stops matching (path rot) or lacks a fixture fails its subtest; stray goldens fail too. Rule PRs are now pure YAML: no Go edits, no shared-golden merge conflicts. Added `CONTEXT.md` (domain glossary) and `docs/plans/2026-07-13-architecture-review.md` (full review, candidates 3–5 parked). Small frictions from the same review: scan cancellation now reaches the filesystem walk (`DirSize`/`discover` honor ctx — a hung network mount can't wedge a scan), `Risk.Actionable()` replaces three inline surface-only checks, and the TUI scroll window has a short-terminal test.

- Rule catalog (Prompt E): 30 rules — every clean.sh module ported plus Tier-S targets. New: DeviceSupport, Archives, CoreSimulator caches/devices/runtimes (simctl-only), docker prune + volumes + VM-disk report, go mod, npm/yarn/pnpm, pip/uv, brew, node_modules discovery, TM snapshots, aerial wallpapers (Sequoia + Tahoe paths), mediaanalysisd cache, user logs, Spotlight reindex, stale macOS installers, /Library/Updates report, Empty Trash, ~/Library/Caches report, iOS backups report. Engine grew: argv-form `native_command`, `note` field for footgun warnings, glob paths, `Host.Root` for fixture-rooted system paths, tool queries (docker df, simctl devices/runtimes, tmutil) with unit-tested parsers. Golden plan covers all 30 rules; `DirSize` treats TCC-denied dirs as partial (du-style), not errors. First real-machine scan: 158.8 GiB found.

- TUI (Prompt D): Bubble Tea checklist — findings grouped by category, size-ranked, risk-colored (safe/caution/surface), regen story + cost for the row under the cursor, safe rules pre-selected, space to toggle, enter opens the plan screen (exact commands, trash destinations, skips with reasons) — nothing executes. `regrow scan` opens the TUI on a TTY; piped or `--json` keeps plain output.
- Rule engine (Prompt C): YAML schema {id, title, category, risk, version-aware per-OS paths, marker discovery, tool_query, native_command, regen, sudo}, strict loader with `--rules-dir` override, du-style size scanner + marker discovery + tool-query registry, dry-run planner emitting the exact command list with path guards and surface-only enforcement. CLI: `regrow scan|plan|rules|version`, `--json`. First 3 rules: xcode-derived-data, go-build-cache, rust-target-dirs. Golden-plan test harness.
- Scaffold: Go module, package layout, CI (macOS test + linux lint).
